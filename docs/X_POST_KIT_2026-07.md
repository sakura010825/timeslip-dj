### なぜリンクを自己リプに置くのか
本文にリンクを入れると、Xはその投稿のリーチを 30〜50% 落とす。
「Xだと伸びない／Threadsのほうが」と言われるものの正体はこれ。
本文は本文だけで完結させ、リンクはぶら下げる。

### リプは3パターンを回して数字で決める
末尾の `-r1 / -r2 / -r3` で landing 側が見分けられる（`/admin` の「流入（landing・UTM）」）。
**1投稿につき1パターン**。同時に並べるとどれが効いたか分からなくなる。

| | 型 | ねらい |
|---|---|---|
| r1 | 宛先だけ | いちばん短い。基準線 |
| r2 | 中身を訴求 | 「このあと何が起きるか」を書く |
| r3 | 自分ごとに寄せる | 「あなたの◯年」と問い返す |

## 投稿後に読む数字

`npm run ops`（redial側）の「2b. 流入（landing・UTM）」に
`x/reply/<セル>-<型>-<r番号>` が出る。出なければ**誰も押していない**。

読む順番:
1. そもそも landing が増えたか（＝リンクが押されたか）
2. r1 / r2 / r3 のどれが押されたか
3. 型A と 型B のどちらが押されたか

3〜5本ぶん回してから比べる。1本では何も分からない。
