# オンデマンド誤読ログ（人名・固有名詞）

オンデマンド生成（ユーザーが選曲して編む一発録りの回）で出た**人名・固有名詞の読み間違い**を、テスター報告や試聴から拾って記録し、`lib/tts-pronunciation-dict.ts` に反映する台帳。

## なぜ台帳が要るか
- オンデマンドは一発録りで、**聴いたその回は直せない**（＝「一発録りの愛嬌」としてUIの免責文でカバー）。
- 代わりに**発音辞書を育てて、次からの回の平均品質を上げる**。公開カタログと違い、オンデマンドは任意の年×季節×曲で未知の固有名詞が出るため、誤読は長い尾を引く。だから「報告→辞書→次から正しく読む」を回し続ける。

## 運用フロー
1. テスター（有料会員）／hideさん試聴で誤読を発見
2. **正しい表記と読みを WebSearch で裏取り**（推測で辞書に入れない・[[feedback_factcheck_before_audio]]）
3. 下の表に1行追加
4. `lib/tts-pronunciation-dict.ts` に `{ pattern: 正しい漢字表記, replacement: 正しいかな, note: ... }` を追加
   - 元スクリプトに出た**正確な漢字表記**をpatternにする（部分一致・リテラル置換のため）
   - 過剰置換を避け、フルネーム等の**一意な表記**に絞る
5. **ワーカーを再起動**して新辞書を反映（[[feedback_retts_needs_fresh_dev_server]]）。以降の生成に効く
6. 反映後、Whisper検査（`scripts/whisper-scan.mjs`）でも検証可能
